# Politica de Seguridad - Ghost-Kali v5.0

**Version:** 5.0.0 | **Fecha:** 2026-06-24

---

## 1. Versiones Soportadas

| Version | Soporte de Seguridad |
|---------|----------------------|
| 5.0.x   | Activo               |
| < 5.0   | Sin soporte          |

## 2. Como Reportar Vulnerabilidades

- **No** abrir un Issue publico para vulnerabilidades de seguridad.
- Enviar el reporte al contacto de seguridad del proyecto
  (ghostkalisecurity@gmail.com) con:
  - Descripcion detallada de la vulnerabilidad.
  - Pasos para reproducir.
  - Impacto potencial.
  - Sugerencia de mitigacion (opcional).
- Respuesta inicial en un maximo de 72 horas.
- Actualizaciones cada 5 dias habiles hasta la resolucion.

## 3. Proceso de Divulgacion Coordinada

1. El reportante y el equipo acuerdan un plazo de divulgacion.
2. Se desarrolla y prueba una correccion.
3. Se publican la correccion y el aviso de seguridad de forma simultanea.
4. Se da credito al reportante, si lo desea.

## 4. Compromisos del Proyecto

- Tomar en serio cada reporte.
- No tomar represalias contra reportantes de buena fe.
- Priorizar la seguridad de las personas usuarias.
- Ser transparente sobre las limitaciones y los riesgos conocidos
  (ver `THREAT_MODEL.md`).

## 5. Hall of Fame

Reconocimiento a quienes han reportado vulnerabilidades de forma responsable.
(Se actualiza conforme se reciben reportes validos.)

## 6. Alcance

Dentro del alcance de esta politica:

- El codigo del repositorio principal.
- Las dependencias directas.
- Las configuraciones por defecto que entrega el toolkit.

Fuera del alcance:

- Infraestructura de terceros (GitHub, Mullvad, la red Tor).
- Configuraciones personalizadas del usuario.
- Ataques de ingenieria social.

---

> La seguridad es un proceso, no un estado. Esta politica se revisa
> periodicamente.
